# R2文章状态机与发布流程

线上网站只读取 `ai/manifest.json`。R2 控制台里的目录只是对象 Key 前缀展示，不是状态来源。

## R2对象布局

```text
ai/
  manifest.json
  manifests/{revision}.json
  state/articles/{slug}.json
  events/{timestamp}-{action}-{slug}.json
  articles/draft/{slug}/index.md
  articles/review/{slug}/index.md
  articles/scheduled/{slug}/index.md
  articles/published/{slug}/index.md
  articles/archived/{slug}/index.md
```

`ai/manifest.json` 是当前线上发布状态。`manifests/` 是历史快照，`state/articles/` 是单篇文章状态，`events/` 是发布事件日志。

## 发布顺序

1. 上传正文到 `ai/articles/published/{slug}/index.md`。
2. 上传图片到 `ai/articles/published/{slug}/images/`。
3. 写入 `ai/state/articles/{slug}.json`。
4. 读取当前 `ai/manifest.json`，生成新的 manifest 内容。
5. 写入不可变快照 `ai/manifests/{revision}.json`。
6. 覆盖 `ai/manifest.json`。
7. 调用网站缓存刷新接口 `/api/articles/cache`。
8. 请求 `/api/articles?action=slugs&site=ai` 验证文章列表。

关键约束：同一时间只允许一个发布任务写 `ai/manifest.json`。内容桥接任务需要串行执行发布。

## 回滚

把某个 `ai/manifests/{revision}.json` 覆盖回 `ai/manifest.json`，然后刷新网站文章缓存。

```bash
node scripts/r2-article-state.mjs rollback \
  --bucket=knowledge-articles \
  --prefix=ai \
  --snapshot=ai/manifests/2026-06-03T10-00-00-000Z.json
```

## 运维命令

验证当前线上状态：

```bash
node scripts/r2-article-state.mjs verify --bucket=knowledge-articles --prefix=ai
```

生成当前线上状态快照：

```bash
node scripts/r2-article-state.mjs snapshot --bucket=knowledge-articles --prefix=ai
```

列出快照：

```bash
node scripts/r2-article-state.mjs list --bucket=knowledge-articles --prefix=ai/manifests/
```

## 生产环境变量

```env
KNOWLEDGE_ARTICLE_R2_SOURCES=[{"site":"ai","source":"web-article","bucket":"knowledge-articles","prefix":"ai","manifestPath":"manifest.json","publicBaseUrl":"https://assets.zgnknowledge.online/ai"}]
KNOWLEDGE_R2_PUBLIC_ASSET_HOST=assets.zgnknowledge.online
KNOWLEDGE_R2_ACCOUNT_ID=4524b1d8551fa19f1fcb2cb6f16ee3b9
```

`KNOWLEDGE_R2_ACCESS_KEY_ID` 和 `KNOWLEDGE_R2_SECRET_ACCESS_KEY` 只放在服务器环境或本地安全环境里，不写入文档和仓库。
