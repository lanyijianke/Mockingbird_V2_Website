# R2提示词媒体迁移与修复

提示词运行时数据仍以 MySQL `Prompts` 表为准。迁移目标是把历史 `/content/prompts/media/*` 文件上传到 R2，并把数据库媒体字段改为公开 R2 URL。新的同步流程只使用临时文件，不再保留本地持久媒体目录；某个媒体项上传失败时直接跳过该项。

## 目标布局

```text
knowledge-articles/
  prompts/media/images/{fileName}
  prompts/media/videos/{fileName}
  prompts/media/previews/{fileName}
  prompts/media/legacy/{fileName}
```

历史媒体迁移规则：

```text
/content/prompts/media/{fileName}
-> https://assets.zgnknowledge.online/prompts/media/legacy/{fileName}
```

新同步规则：

```text
KNOWLEDGE_PROMPT_MEDIA_R2_BUCKET=knowledge-articles
KNOWLEDGE_PROMPT_MEDIA_R2_PREFIX=prompts/media
KNOWLEDGE_PROMPT_MEDIA_R2_PUBLIC_BASE_URL=https://assets.zgnknowledge.online/prompts/media
```

## 停止条件

出现以下任一情况，停止迁移，不要执行数据库改写：

- `missingLocalFiles > 0` 且这些文件被线上 DB 行引用。
- audit 文件无法写入。
- R2 凭据或 bucket 配置缺失。
- `upload` 有失败项。
- `verify-r2` 有 missing 或 content-length mismatch。
- `backup-db` 失败或备份文件为空。
- `dry-run-db` 的行数明显不符合预期。

## 迁移顺序

在容器内执行，路径按生产部署调整：

```bash
node scripts/prompt-media-r2-migrate.mjs audit --media-dir=/opt/mockingbird-knowledge-web/media --out=/tmp/prompt-media-audit.json
node scripts/prompt-media-r2-migrate.mjs upload --audit=/tmp/prompt-media-audit.json --dry-run
node scripts/prompt-media-r2-migrate.mjs upload --audit=/tmp/prompt-media-audit.json
node scripts/prompt-media-r2-migrate.mjs verify-r2 --audit=/tmp/prompt-media-audit.json
node scripts/prompt-media-r2-migrate.mjs backup-db --out=/tmp/prompts-media-backup-2026-06-03.json
node scripts/prompt-media-r2-migrate.mjs dry-run-db --backup=/tmp/prompts-media-backup-2026-06-03.json
node scripts/prompt-media-r2-migrate.mjs apply-db --backup=/tmp/prompts-media-backup-2026-06-03.json
```

迁移完成后抽样验证：

```bash
curl -I "https://assets.zgnknowledge.online/prompts/media/legacy/<sample-file>"
curl -fsS "https://zgnknowledge.online/api/prompts?page=1&pageSize=20" >/tmp/prompts-api.json
```

## 回滚

如果数据库 URL 改写后运行时异常，用备份文件回滚：

```bash
node scripts/prompt-media-r2-migrate.mjs rollback-db --backup=/tmp/prompts-media-backup-2026-06-03.json
```

回滚只会更新当前值仍匹配迁移后 R2 URL 的行；如果某些行在迁移后被业务同步改过，脚本会跳过并报告 skipped。

## Seedance视频修复

新 README adapter 已恢复 Seedance 视频识别。`YouMind-OpenLab/awesome-seedance-2-prompts` 是视频模型提示词源；新版 README 可能只提供 YouMind 观看链接和 X/Twitter 缩略图，不能把这类记录判定为“仅图片”。同步器的解析顺序应是：

1. README 里的直接 `.mp4`。
2. Cloudflare Stream 缩略图反推 `/downloads/default.mp4`。
3. YouMind `/youmarketing-api/video-prompts` 元数据里的 `caption` mp4 或 `streamId`。
4. 如果 YouMind 元数据无法命中当前 watch id，回落到 README 的原始 X/Twitter `SourceUrl`，后续用 `yt-dlp` 下载。

历史缺失视频用独立脚本修复：

```bash
node scripts/prompt-video-repair.mjs audit --source=yoomind-seedance-2 --out=/tmp/seedance-video-repair-audit.json
node scripts/prompt-video-repair.mjs dry-run --audit=/tmp/seedance-video-repair-audit.json
node scripts/prompt-video-repair.mjs apply --audit=/tmp/seedance-video-repair-audit.json --revalidate
```

脚本只更新 `VideoPreviewUrl` 为空的行，不覆盖已有视频。卡片预览依赖容器内 `ffmpeg`，生成失败时只补全量视频并在 `/tmp/seedance-video-repair-report.json` 记录。

### X/Twitter来源批量回填

如果 audit 发现大量 active `seedance-2` 行 `SourceUrl` 是 X/Twitter，但 `VideoPreviewUrl` 为空，按小批次推进，避免反复卡在失败项：

```bash
node scripts/prompt-video-repair.mjs audit-missing-x --limit=100 --out=/tmp/seedance-missing-x-batch-001.json
node scripts/prompt-video-repair.mjs apply --audit=/tmp/seedance-missing-x-batch-001.json --revalidate
```

如果某批有失败项，下一批用上一批 audit 的 `lastId` 往后推进：

```bash
node scripts/prompt-video-repair.mjs audit-missing-x --after-id=<lastId> --limit=100 --out=/tmp/seedance-missing-x-batch-002.json
node scripts/prompt-video-repair.mjs apply --audit=/tmp/seedance-missing-x-batch-002.json --revalidate
```

需要逐个刷新详情页时加：

```bash
node scripts/prompt-video-repair.mjs apply \
  --audit=/tmp/seedance-missing-x-batch-002.json \
  --revalidate \
  --revalidate-details \
  --site-url=https://zgnknowledge.online
```

要求运行环境里存在正确的 `KNOWLEDGE_ADMIN_TOKEN` 或 `ADMIN_API_TOKEN`。如果 token 不匹配，`/tmp/seedance-video-repair-report.json` 的 `revalidation` 会记录失败；此时数据库可能已修好，但前端页面仍可能看到旧缓存。

### GitHub旧锚点行

旧导入数据里可能存在 `SourceUrl LIKE 'https://github.com/%'` 的 `seedance-2` 行。这类行不能直接按 GitHub URL 当视频源修复；必须先做标题匹配审计：

```bash
node scripts/prompt-video-repair.mjs audit-missing-github-title --out=/tmp/seedance-missing-github-title-audit.json
```

只允许修复 `matches` 中按当前 README `RawTitle` 精确匹配、且 `videoUrl` 明确存在的行。`unmatchedRows` 需要旧版 README 或旧提交溯源，不要猜测视频 URL。

### 修复后核对

数据库核对：

```bash
node --input-type=module - <<'NODE'
import fs from 'node:fs';
import mysql from 'mysql2/promise';
function loadEnv(file) {
  if (!fs.existsSync(file)) return;
  for (const line of fs.readFileSync(file, 'utf8').split(/\r?\n/)) {
    const t = line.trim();
    if (!t || t.startsWith('#') || !t.includes('=')) continue;
    const i = t.indexOf('=');
    const k = t.slice(0, i).trim();
    let v = t.slice(i + 1).trim();
    if ((v.startsWith('"') && v.endsWith('"')) || (v.startsWith("'") && v.endsWith("'"))) v = v.slice(1, -1);
    process.env[k] ||= v;
  }
}
loadEnv('.env.local');
loadEnv('.env');
const conn = await mysql.createConnection({ uri: process.env.MYSQL_URL, charset: 'utf8mb4' });
const [summary] = await conn.query(`
  SELECT
    COUNT(*) total,
    SUM(IsActive = 1) active,
    SUM(IsActive = 1 AND VideoPreviewUrl IS NOT NULL AND VideoPreviewUrl <> '') withVideo,
    SUM(IsActive = 1 AND (VideoPreviewUrl IS NULL OR VideoPreviewUrl = '')) missingTotal,
    SUM(IsActive = 1 AND (VideoPreviewUrl IS NULL OR VideoPreviewUrl = '') AND SourceUrl REGEXP '^https?://(www\\\\.)?(x|twitter)\\\\.com/') missingX,
    SUM(IsActive = 1 AND (VideoPreviewUrl IS NULL OR VideoPreviewUrl = '') AND SourceUrl LIKE 'https://github.com/%') missingGithub
  FROM Prompts
  WHERE Category = 'seedance-2'
`);
console.log(JSON.stringify(summary, null, 2));
await conn.end();
NODE
```

API 核对：

```bash
curl -fsS "https://zgnknowledge.online/api/prompts?page=1&pageSize=20&category=seedance-2" | jq '.data.items[] | {id,title,videoPreviewUrl,cardPreviewVideoUrl}'
curl -fsS "https://zgnknowledge.online/api/prompts/<promptId>" | jq '.data | {id,title,videoPreviewUrl,cardPreviewVideoUrl}'
```

页面核对：

- 列表卡片默认显示封面图；桌面端鼠标 hover 后才播放 `CardPreviewVideoUrl || VideoPreviewUrl`。
- 移动端/触控设备不会自动播放卡片预览，只显示播放徽标。
- 详情页应直接显示 `<video controls>`。如果 API 已有视频但详情页仍显示“仅封面图”，优先检查是否部署了包含 `app/ai/prompts/[id]/page.tsx` stale-cache 修复的版本，并触发 revalidate 或重新部署。

### 缓存与部署注意事项

直接改 MySQL 不会自动刷新 Next 页面缓存。批量回填后必须满足至少一项：

- `apply` 时带 `--revalidate`，必要时带 `--revalidate-details`。
- 用正确线上 admin token 调用 `/api/revalidate/content`。
- 重新部署一次，清掉旧进程里的页面状态。

曾经出现过的故障：详情页模块级 `promptDetailLoads` Map 会把某个 prompt 第一次读取到的 DB 结果常驻在 Next 进程里，导致 DB/API 已经有视频，但详情页仍显示旧的“仅封面图”。该问题修复后，不要再在 App Router 页面模块顶层缓存 DB 行或 Promise；ISR/`revalidatePath` 应作为页面缓存层。
